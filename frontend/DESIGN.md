# LiveBrief Frontend — 디자인 시스템

LiveBrief 프론트엔드("The Verification Desk")의 디자인 문서.
컨셉은 **에디토리얼 뉴스룸**이다: 신문 조판의 규율(헤어라인, 대문자 키커, 세리프
헤드라인)을 화면에 옮기되, 검증이라는 행위를 물리적인 카드 다루기(집기, 넘기기,
도장 찍기)로 표현한다. 이모지·그라데이션·글래스모피즘 등 "AI가 만든 느낌"의
장식은 전면 배제한다.

---

## 1. 디자인 원칙

1. **종이와 잉크.** 배경은 종이(웜 오프화이트), 글자는 잉크(웜 블랙). 색은
   의미가 있을 때만 쓴다 — 빨강은 속보/경고, 초록은 승인, 앰버는 보류.
2. **판단은 사람이 한다.** 백엔드 철학(AI는 추출만, 진위 판단은 기자)을 UI가
   그대로 드러낸다. 모든 상태 변화는 사용자의 명시적 행동(스와이프, 도장)에서만
   발생한다.
3. **타이포그래피가 곧 UI다.** 아이콘 대신 활자 위계(세리프/산세리프/모노)로
   정보를 구분한다. 이모지 금지.
4. **숫자는 데이터답게.** 시각·수치·카운트는 전부 모노스페이스 + tabular figures.
5. **모션은 물리적 은유로만.** 카드가 날아오고, 들리고, 떨어진다. 장식적 모션은
   넣지 않는다. `prefers-reduced-motion` 전역 대응.

## 3. 타이포그래피

Google Fonts 3종. 데이터는 영어이므로 라틴 조판 기준.

| 역할 | 폰트 | 사용처 |
| --- | --- | --- |
| Display/본문 세리프 | **Newsreader** (variable, optical sizing) | 마스트헤드, 클레임 헤드라인, 브리프 본문, 근거 인용 |
| UI 산세리프 | **Libre Franklin** | 버튼, 키커, 라벨, 탭 — 대문자 + 자간 0.1em 내외 |
| 데이터 모노 | **IBM Plex Mono** | 시각, 카운트(`01 / 08`), 신뢰도 %, 와이어 티커 |

관례:
- 키커(`.kicker`)는 항상 대문자 + `letter-spacing: 0.18em` + 와이어 레드.
- 마스트헤드 룰(`.masthead-rule`)은 신문식 이중선(3px + 1px).
- 근거 인용은 이탤릭 세리프 + 좌측 3px 헤어라인.

## 4. 화면 흐름

```
Landing ─→ Analyzing ─→ Desk ─→ BriefView(저장된 deck brief)
   └→ empty Desk ─→ UrlStoryDialog ─┘    ↑└─ Back to the desk
```

| 화면 | 파일 | 요지 |
| --- | --- | --- |
| Landing | `screens/Landing.jsx` | 마스트헤드, 라이브 시계, 하단 와이어 티커(마키), 3단 기능 소개 |
| Analyzing | `screens/Analyzing.jsx` | 파이프라인 5단계 체크리스트 연출 + 실제 API 호출 병행 |
| Desk | `screens/Desk.jsx` | 검증 데스크, 빈 보드, URL 추가 진입 — 아래 5절 |
| UrlStoryDialog | `screens/UrlStoryDialog.jsx` | URL-only 크롤링 미리보기 확인 후 분석/추가 |
| BriefView | `screens/BriefView.jsx` | deck별 저장된 디스패치: 타자기 렌더링, 사용/제외/보류 수치, 세이프티 노트 |

데모는 `api.js`가 번들된 목 케이스(`mock.js`)를 사용한다. 커스텀 URL 추가는
백엔드 `/analysis/preview`로 먼저 크롤링/정제 결과를 확인한 뒤, `/analysis/route`로
기존 스토리 덱에 붙일지 새 덱을 만들지 결정한다. 확인된 본문은 `/analysis/run`으로
보낸다. 브리프 생성은 API 실패 시 동일한 무발명(no-invention) 규칙의 로컬 컴포저로
폴백.

## 5. Desk 레이아웃

### 데스크톱 (≥1024px) — 3패널 그리드

```
┌────────────┬──────────────────────┬────────────┐
│ 소스 온 파일 │        STAGE         │ 진행 드래프트 │
│ 컨플릭트 알림 │  케이스 보드 ↔ 기사 카드 덱 │  데스크 탤리  │
└────────────┴──────────────────────┴────────────┘
```

상단: 토픽 탑바 + 4px 진행 레일(전체 클레임 대비 판정 비율, 와이어 레드).

### 모바일 (<1024px)

- 좌우 레일 숨김 → 하단 고정 시트 탭(`Draft (n)` / `Sources (n)` / `n conflicts`)으로 대체. 탭은 슬라이드업 시트(`.sheet`)를 연다.
- 카드는 콘텐츠 기반 높이(top 카드 `position: relative`, 뒷장은 absolute)로
  전환해 잘림을 방지. 컨트롤은 카드 바로 아래.
- `100dvh`, `env(safe-area-inset-bottom)` 사용.

## 6. 케이스 보드 (토픽 클러스터)

`screens/ClusterField.jsx`. 유사한 주제의 기사들을 **스토리 덱(pile)** 으로
스테이지에 흩어 놓는다. 현재 라우팅은 AI가 담당하고, 추후 텍스트 임베딩 모델 API로
교체할 수 있게 백엔드 `ArticleDeckRouter`에 모아 둔다.

- **배치**: 미리 정의된 8개 슬롯(% 좌표)에 순서대로. 더미당 `--tilt`
  (-2.4°~2.4°)로 손으로 놓은 느낌.
- **부유**: `floaty` 키프레임(6.5~10초, 더미별 위상차)으로 위아래 ±6px 드리프트.
- **더미 구성**: 뒤에 어긋난 종이 2장(`.pile-sheet`) + 앞장(라벨 키커, `n / m
  ruled` 카운트, contested 뱃지, 하단 진행 바).
- **완료 상태**: 초록 "RULED" 스탬프(−10° 회전) + 앞장 62% 불투명 + 부유 정지.

### 모션 시퀀스

| 이름 | 트리거 | 내용 |
| --- | --- | --- |
| `deal-in` | 보드 최초 진입(세션당 1회) | 더미들이 위(-38vh)에서 120ms 간격 스태거로 낙하·감속 착지 후 부유 시작 |
| pick-up | 더미 클릭 | 선택 더미 1.14배 리프트 + 그림자 확대, 나머지는 220ms 페이드아웃 → 240ms 뒤 덱으로 전환 |
| `deck-grow` | 덱 마운트 | 클릭한 더미의 실좌표(`--from-x/y`)에서 scale 0.24 → 1.0, 미세 오버슈트 이징 |
| settle-back | 더미의 마지막 기사 카드 판정 | "DECK RULED — RETURNING TO THE BOARD" 600ms 후 보드 복귀 |

## 7. 기사 카드와 스와이프

`screens/ClaimCard.jsx`. 파일명은 유지하지만 실제 카드 단위는 **기사 1개**다.
Pointer Events 기반 자체 드래그 물리.

- **드래그**: `translate(dx, dy) rotate(dx/16deg)`, 드래그 중 transition 해제.
- **임계값**: 가로 100px(승인/기각), 세로 아래 96px(보류, 수직 우세일 때만).
- **스탬프**: 드래그 방향/거리에 비례해 러버스탬프 오버레이 표시 —
  `APPROVED`(초록, 좌상단 −9°) / `DISCARDED`(빨강, 우상단 9°) /
  `HOLD — VERIFY`(앰버, 하단 −3°).
- **퇴장**: 판정 방향으로 300ms 비행(`120vw` / `-120vw` / `110vh`) 후 커밋.
- **명령 통일**: 버튼·키보드 판정은 `command {status, seq, id}`를 톱카드에
  내려보내 스와이프와 동일한 퇴장 애니메이션을 태운다. `id`로 대상 카드를
  고정해 연쇄 오작동을 방지(중요).
- **언두**: 판정 스택 pop + 복귀 카드가 판정 방향 반대에서 날아 들어오는
  `card-return-*` 키프레임. 덱 안에서는 해당 클러스터의 판정만 되돌릴 수 있고,
  보드/완료 화면에서 언두하면 해당 클러스터 덱이 자동으로 다시 열린다.
- **키보드 맵**: `→` 승인 · `←` 기각 · `↓` 보류 · `z`/`u`/`Backspace` 언두 ·
  `Esc` 보드로 복귀.
- **카드 구조** (위→아래): Article on file 키커 + `nn / mm` 카운트 →
  리스크/claim count/contested 뱃지 → 출처명 → 정제된 기사 본문 발췌 →
  이 기사에서 추출된 claim 목록 → (있으면) 이 기사와 연결된 컨플릭트 스트립 →
  푸터(스토리명·타입 / 평균 추출 신뢰도 카운트업 % + 3밴드 컬러 미터).

## 8. 마이크로 인터랙션 목록

| 요소 | 효과 |
| --- | --- |
| LIVE 점 | 1.6초 펄스 |
| 라이브 시계 | 초 단위 갱신, 초는 소프트 잉크 |
| 와이어 티커 | 잉크 배경 마키 36초 루프, `+++` 구분자 |
| 랜딩 요소 | `settle`(14px 상승 페이드) 스태거 진입 |
| 신뢰도 미터 | 마운트 시 0→값 채움(0.8s), 숫자는 cubic ease-out 카운트업 |
| 진행 레일 | width 0.4s 트랜지션 |
| 드래프트 라인 | 추가 시 `settle` + 카운터 번호(`01`, `02`…) |
| 완료 수치 | 카운트업 |
| 디스패치 본문 | 타자기(14ms/자) + 빨간 캐럿, 완료 후 메타 영역 슬라이드업 |
| 버튼 호버 | 프라이머리→와이어 레드, 판정 버튼→각 판정색 반전 |

이징 공통 토큰: 진입/정착은 `cubic-bezier(0.2, 0.7, 0.2, 1)`, 퇴장은
`cubic-bezier(0.5, 0, 0.9, 0.4)`, 덱 확대는 미세 오버슈트
`cubic-bezier(0.24, 0.9, 0.3, 1.04)`.

## 10. 파일 맵

```
frontend/
  index.html            폰트 로드, 루트
  src/
    styles.css          전체 디자인 시스템(토큰·컴포넌트·모션·반응형)
    api.js              API 클라이언트 + 오프라인 폴백 + 로컬 브리프 컴포저
    mock.js             번들 데모 케이스(AnalysisResult 계약 준수)
    bits.jsx            CountUp · Meter · RiskBadge · Clock · Typewriter
    App.jsx             화면 상태 머신
    screens/
      Landing.jsx       마스트헤드 + 티커
      Analyzing.jsx     파이프라인 연출 + 실제 호출
      Desk.jsx          검증 데스크(판정 상태 소유)
      UrlStoryDialog.jsx URL-only 크롤링 미리보기 + 분석 추가
      ClusterField.jsx  케이스 보드(토픽 더미)
      ClaimCard.jsx     기사 스와이프 카드(드래그 물리)
      BriefView.jsx     저장된 deck brief 재열람
```

상태 소유권: 기사 카드 판정(`verdicts`, source id → status)과 판정 이력(`history`)은
전부 **Desk**가 갖는다. 백엔드에는 상태 저장 API가 없으므로(무상태), 브리프 생성 시
승인된 기사에 속한 claim만 `POST /analysis/brief`로 보낸다.

## 11. 하지 말 것

- 이모지, 장식용 아이콘, 그라데이션 배경, 보라색 계열.
- 판정을 자동으로 내리는 어떤 UI도 금지(기본 선택값 포함).
- 색만으로 의미 전달 금지 — 뱃지에는 항상 텍스트 라벨 병기.
- 카드 밖으로 나가는 가로 스크롤. 넘치는 표/코드는 자체 스크롤 컨테이너에.
